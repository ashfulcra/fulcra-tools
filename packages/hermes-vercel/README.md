# hermes-vercel

The Fulcra "press play" onboarding demo. One command spins up an isolated
Hermes-agent sandbox on Vercel Sandbox for a guest; you send them the URL,
they land in a locked-down chat that walks them through connecting their data
to Fulcra.

## New here? Start with the handoff doc.

**→ [`docs/HANDOFF.md`](docs/HANDOFF.md)** is the full report: problem
statement, architecture, component-by-component breakdown, cost model,
limitations, runbook, and the open decisions for whoever owns this next.

## Quick reference

```bash
# First time: bake the snapshot (3–5 min)
npm install
cp .env.example .env  # then fill in VERCEL_* + OPENROUTER_*
npm run build

# Spawn a guest sandbox (~15s, prints a URL)
npm run spawn alice -- --mode friendly
# or: --mode marketing

# Clean up
npm run teardown -- --all
```

## Sibling repos

- [`litellm`](../litellm) — the LLM gateway that mints
  per-sandbox virtual keys (caps + TTL). Deployed at
  `https://fulcra-litellm.fly.dev`. Wire it via `LITELLM_URL` +
  `LITELLM_MASTER_KEY` in `.env` to flip from the baseline key-injection
  path to the gateway path.
- [`ashfulcra/fulcra-hermes-daytona`](https://github.com/ashfulcra/fulcra-hermes-daytona)
  — the original Daytona port. **Deprecated** (Daytona's egress allowlist blocks
  `api.fulcradynamics.com`) and extracted out of this monorepo into its own
  standalone (archived) repo. Kept as a reference. See `docs/HANDOFF.md` § 8.4.

## File map

```
src/
  build-snapshot.js   one-time: provisions a base sandbox, installs the stack, snapshots it
  spawn.js            per-guest: spawn from snapshot, mint key, boot chat, print URL
  teardown.js         cleanup (per sandbox or --all)
  config.js           .env loader
  fetch-shim.js       Node 26 / undici brotli workaround (see HANDOFF § 8.5)
assets/
  hermes/start-chat.sh   detached boot inside the sandbox
  hermes/AGENTS.md       ambient context for the agent
  hermes/SOUL.md         agent persona
  caddy/Caddyfile        the lockdown deny list
docs/
  HANDOFF.md          ← read this first
  ARCHITECTURE.md     earlier design doc; overlaps with HANDOFF in parts
.snapshot.json        current snapshot ID (gitignored; regenerated by `npm run build`)
.env / .env.example   operator secrets (.env gitignored)
```
